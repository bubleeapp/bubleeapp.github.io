Pequena Explicação.

Viemos revolucionar a comunicação < Estamos construindo um mensageiro a partir de várias tecnologias open source, mas não um cliente do Telegram (httpd://telegram.org) e sim um mensageiro que servidores e estrutura física baseados no Brasil. O Bublee terá seus servidores totalmente nacionais rodando em Data Centers do nosso pais, isso pode ser bom?? talvez! visto a qualidade dos serviços de internet fornecidos a nós do Brasil poderemos ter algumas dificuldades em fazer com que seja totalmente viável ter todo o serviço localizado no país mas estamos trabalhando através de acordos comerciais e etc para que isso ocorra.

"Plataforma do Bublee é preparada para lidar com milhões de usuários simultâneos. Esta documentação(em breve) explicará detalhes técnicos sobre a plataforma e sobre o seu uso.

O protocolo para comunicação entre o servidor e os clientes que foi inspirado por MTProto de Telegram e nomeado após ele: MTProto v2.

O servidor é implementado em cima do framework de alto desempenho Akka usa o PostgreSQL para persistência.

As aplicações são separado para o núcleo e UI. Core é uma única base de código escrito em Java 6, sem reflexão e depois convertido em vários idiomas. UI é implementada com ferramentas nativas para cada plataforma. Nós fizemos alguns testes e fundou que esta tão rápido quanto implementações nativas."
